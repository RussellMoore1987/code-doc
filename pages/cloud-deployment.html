Cloud Deployment

DevDocs can be deployed to any static hosting platform or cloud provider. This page covers deployment configurations for the most popular options, from simple CDN hosting to enterprise cloud environments.

ℹ Choosing a Platform

For personal and small team projects, GitHub Pages, Netlify, or Cloudflare Pages are the fastest path. For enterprise deployments with compliance requirements, use your organization's preferred cloud provider (AWS S3 + CloudFront, Azure Static Web Apps, or Google Cloud Storage + Cloud CDN).

Netlify

Netlify supports zero-config deployments from a Git repository:

  1. Connect your repository at app.netlify.com.
  2. Set the publish directory to . (the project root).
  3. Leave the build command empty (no build step needed).
  4. Click Deploy site.
# netlify.toml — optional configuration
[build]
  publish = "."

[[headers]]
  for = "/*"
  [headers.values]
    X-Frame-Options = "DENY"
    X-Content-Type-Options = "nosniff"
    Referrer-Policy = "strict-origin-when-cross-origin"

[[headers]]
  for = "/*.html"
  [headers.values]
    Cache-Control = "no-cache"

[[headers]]
  for = "/*.js"
  [headers.values]
    Cache-Control = "public, max-age=31536000, immutable"

AWS S3 + CloudFront

S3 Bucket Setup

# Create bucket
aws s3 mb s3://my-devdocs --region us-east-1

# Enable static website hosting
aws s3 website s3://my-devdocs \
  --index-document index.html \
  --error-document index.html

# Sync files
aws s3 sync . s3://my-devdocs \
  --exclude ".git/*" \
  --exclude "*.md" \
  --cache-control "no-cache" \
  --delete

CloudFront Distribution

# Create distribution (simplified)
aws cloudfront create-distribution \
  --origin-domain-name my-devdocs.s3-website-us-east-1.amazonaws.com \
  --default-root-object index.html

Cloudflare Pages

# Using Wrangler CLI
npm install -g wrangler

wrangler pages project create devdocs
wrangler pages deploy . --project-name devdocs

Cloudflare Pages automatically provisions a global CDN, HTTPS, and DDoS protection.

Azure Static Web Apps

# staticwebapp.config.json
{
  "navigationFallback": {
    "rewrite": "/index.html"
  },
  "globalHeaders": {
    "X-Content-Type-Options": "nosniff",
    "X-Frame-Options": "DENY"
  },
  "mimeTypes": {
    ".html": "text/html",
    ".js": "application/javascript",
    ".css": "text/css"
  }
}

Google Cloud Storage

# Create bucket and enable public access
gsutil mb gs://my-devdocs
gsutil iam ch allUsers:objectViewer gs://my-devdocs
gsutil web set -m index.html gs://my-devdocs

# Upload
gsutil -m rsync -r -d . gs://my-devdocs

Custom Domain Configuration

Point your domain to the hosting platform and configure HTTPS. Most platforms provision TLS certificates automatically via Let's Encrypt. 

# Example DNS records (adjust for your provider)
# CNAME for subdomain
docs     CNAME   my-project.netlify.app.

# A records for apex domain
@        A       76.76.21.21
@        A       76.76.21.22
ℹ HTTPS Required

DevDocs loads page fragments via the Fetch API, which requires a secure context (https://) in most browsers. Always enable HTTPS in production, even if the content is public.
