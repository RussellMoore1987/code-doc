Environment Setup

This guide covers how to configure your development environment for optimal DevDocs development. Whether you're customizing the theme, adding new features, or preparing to contribute, these environment configurations will streamline your workflow.

Essential Development Tools

While DevDocs runs with just a basic HTTP server, having the right development tools makes content creation and customization much more efficient.

Code Editor Configuration

We recommend Visual Studio Code with these extensions:

  • Live Server - Instant reload when you save HTML/CSS/JS changes
  • HTML CSS Support - Better autocomplete for CSS classes
  • Emmet - Rapid HTML scaffolding (usually built-in)
  • Prettier - Code formatting for consistent style
💡 Pro Tip

Enable Format on Save in VS Code settings to automatically clean up your HTML and CSS as you work. This keeps the codebase consistent across team members.

Browser Developer Tools

Since DevDocs uses extensive CSS custom properties and JavaScript interactions, having proper browser tools configured is essential:

  • Chrome DevTools - Best CSS Grid/Flexbox inspection
  • Firefox Developer Edition - Excellent CSS custom property debugging
  • Accessibility Inspector - Built into both browsers, crucial for testing navigation

Understanding Project Structure

DevDocs follows a flat file structure that keeps everything organized without complex build systems: 

devdocs/
├── index.html          # Main application shell
├── styles.css          # All styling + CSS custom properties
├── app.js             # Navigation, search, UI interactions
├── images/            # Logos, icons, diagrams
│   ├── logo.svg
│   └── dark_logo.svg
└── pages/             # Content fragments (loaded via Ajax)
    ├── intro.html
    ├── installation.html
    ├── first-steps.html
    ├── environment-setup.html
    └── ...

Content Authoring Workflow

The typical workflow for adding new content:

  1. Create a new HTML file in the pages/ directory
  2. Structure content using proper heading hierarchy (h1 → h2 → h3 → h4)
  3. Add the page reference to NAV_DATA in app.js
  4. Test navigation, search, and right-sidebar outline generation
  5. Validate accessibility with screen reader testing

Customization Environment

If you plan to customize DevDocs beyond content changes, these environment setups will help you work more effectively with the codebase.

CSS Custom Properties Workflow

All theming happens through CSS custom properties. Create a theme-dev.css file during development to test color schemes without modifying the main stylesheet: 

/* theme-dev.css - development overrides */
:root {
  --accent: #ff6b35;        /* Test orange theme */
  --accent-light: #fff4f2;
  --nav-active-bar: #ff6b35;
  --bg-header: #1a1a1a;     /* Dark header experiment */
}

[data-theme="dark"] {
  --accent: #ff8c69;
  --bg-header: #0d1117;
}

Link this file after styles.css in your local index.html for testing.

JavaScript Debugging Setup

Enable JavaScript debugging features in your browser console. DevDocs logs helpful information that aids in navigation and search troubleshooting:

  • Navigation tree build process and timing
  • Ajax content loading success/failure
  • Search index creation and query performance
  • Scroll-spy and outline generation events
🔧 Debug Mode

Add ?debug=true to your localhost URL to enable verbose console logging. This reveals the internal state changes as you navigate through the documentation.

Performance Monitoring

Even though DevDocs is lightweight, monitoring performance during development ensures your content additions don't degrade the user experience.

Lighthouse Integration

Run regular Lighthouse audits to check:

  • Performance - JavaScript execution time, content loading speed
  • Accessibility - Keyboard navigation, screen reader compatibility
  • Best Practices - HTTPS usage, console errors, deprecated APIs
  • SEO - Meta descriptions, heading hierarchy, link text

Content Size Monitoring

Keep individual page files under 50KB for optimal loading performance. Use browser DevTools to monitor:

  • Network tab → Ajax requests for page content
  • Performance tab → JavaScript execution during navigation
  • Memory tab → DOM node count as content grows

Testing Environment

Comprehensive testing ensures DevDocs works reliably across different browsers and devices.

Cross-Browser Testing

Test core functionality across these minimum browser requirements:

  • Chrome 90+ - CSS Grid, custom properties, Fetch API
  • Firefox 85+ - JavaScript modules, CSS :focus-visible
  • Safari 14+ - CSS aspect-ratio, ResizeObserver
  • Edge 90+ - Same as Chrome (Chromium-based)

Responsive Design Testing

Test the responsive behavior at these key breakpoints using browser DevTools device emulation:

  • Mobile - 375px width (sidebar collapses to overlay)
  • Tablet - 768px width (right sidebar may be hidden)
  • Desktop - 1200px+ width (full three-column layout)
📱 Mobile Testing

Pay special attention to touch interactions on mobile devices. The sidebar resize handles and navigation should work smoothly with touch gestures, not just mouse interactions.
