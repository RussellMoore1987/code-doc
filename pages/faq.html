Getting Started

What is DevDocs?

DevDocs is a lightweight, static documentation platform built with vanilla HTML, CSS, and JavaScript. It features a three-column layout with collapsible navigation, real-time search, and responsive design. Unlike complex documentation generators, DevDocs requires no build step or framework dependencies.

Do I need Node.js or any build tools?

No! DevDocs runs entirely in the browser with static files. You only need:

  • A basic HTTP server (VS Code Live Server, Python's `http.server`, or any web server)
  • A modern web browser that supports ES6+ JavaScript
  • A text editor for creating content

The optional CLI tools require Node.js, but they're not necessary for basic usage.

How do I add new pages?

Adding new pages is a two-step process:

  1. Create the HTML file in the `pages/` directory
  2. Add it to navigation by updating the `NAV_DATA` array in `app.js`

The search index will automatically include new pages when it rebuilds.

Can I use Markdown instead of HTML?

DevDocs uses HTML by design for maximum control and performance. However, you can:

  • Convert existing Markdown with tools like Pandoc
  • Use the optional CLI converter: `devdocs convert markdown`
  • Write Markdown and manually convert to HTML as needed

Customization

How do I change colors and theming?

DevDocs uses CSS custom properties for theming. Override them in your own CSS file or add a <style> block to index.html: 

:root {
  --accent: #your-primary-color;
  --accent-light: #your-light-color;
  --bg-header: #your-header-bg;
}

[data-theme="dark"] {
  --accent: #your-dark-theme-primary;
}

Can I modify the three-column layout?

Yes, but it requires CSS modifications. The layout uses CSS Grid and custom properties for sizing. Common modifications include:

  • Hide right sidebar - Set `--right-sidebar-width: 0`
  • Change default widths - Modify width constants in `app.js`
  • Add more columns - Extend the CSS Grid template areas

How do I add my own logo and branding?

Replace the logo files in the `images/` directory and update the references in `index.html`: 

<div class="header-brand">
  <img src="/images/your-logo.svg" alt="Your Logo" class="logo">
  <span class="brand-title">Your Project</span>
</div>

Update the page title, footer, and any other branding elements in the same file.

Content Management

How big can individual pages be?

For optimal performance, keep pages under 50KB. Pages over 100KB may cause noticeable loading delays. Consider splitting large pages into multiple smaller pages for better user experience. **Search Test**

How should I organize my navigation structure?

Follow these best practices:

  • Maximum 4 levels deep - Deeper nesting becomes hard to navigate
  • 7±2 items per section - Based on cognitive load principles
  • Logical grouping - Group related concepts together
  • Progressive disclosure - Start with basics, progress to advanced

Can I include images and media?

Yes! Place images in the `images/` directory and reference them in your HTML: 

<!-- Standard image -->
<img src="/images/diagram.png" alt="Architecture diagram">

<!-- Responsive image with DevDocs styling -->
<figure class="img-block">
  <img src="/images/screenshot.png" alt="Interface screenshot">
  <figcaption>Figure 1 - Main interface</figcaption>
</figure>

For best performance, optimize images and consider using SVG for diagrams and icons.

Search

How does the search functionality work?

DevDocs builds a client-side search index by fetching all pages and extracting text content. The search index includes:

  • Page titles (highest weight)
  • Heading text (medium weight)
  • Body content (lower weight)
  • Section context for better results

Why isn't search working?

Common search issues and solutions:

  • CORS errors - Make sure you're using an HTTP server, not opening files directly
  • Empty results - Check browser console for fetch errors or navigation data issues
  • Slow search - Large sites may need search optimization or pagination

How can I improve search results?

Improve search by:

  • Using descriptive headings - Clear, keyword-rich headings rank higher
  • Adding meta content - Include relevant keywords in introductory paragraphs
  • Structuring content well - Proper heading hierarchy improves context
  • Avoiding duplicate content - Each page should have unique, focused content

Performance

My documentation site is loading slowly. How can I fix this?

Common performance fixes:

  • Optimize images - Compress PNGs, use WebP, or switch to SVG for graphics
  • Split large pages - Break content over 50KB into multiple pages
  • Enable HTTP caching - Configure proper cache headers on your server
  • Check network requests - Use browser DevTools to identify slow requests

I have hundreds of pages. Will performance suffer?

DevDocs is designed to handle large documentation sites. Performance optimizations include:

  • Lazy search indexing - Index builds only when needed
  • Efficient navigation - Virtual scrolling for large navigation trees
  • Memory management - Content is loaded on-demand and cached intelligently

Sites with 200+ pages should consider the optional build optimizations for production.

Deployment

Where can I host my DevDocs site?

DevDocs works on any static hosting platform:

  • GitHub Pages - Free, automatic deployments from git repos
  • Netlify - Easy drag-and-drop deployment with preview builds
  • Vercel - Fast global CDN with branch previews
  • AWS S3 - Scalable hosting with CloudFront CDN
  • Traditional web hosting - Any host that serves static files

Do I need HTTPS?

While DevDocs works over HTTP, HTTPS is recommended (and required) for:

  • Modern browser features and security
  • SEO benefits and browser trust indicators
  • Service worker functionality (if added)
  • Integration with analytics and external services

Most static hosting platforms provide free HTTPS automatically.

Browser Support

Which browsers are supported?

DevDocs supports all modern browsers:

  • Chrome 90+ (and Chromium-based browsers)
  • Firefox 85+
  • Safari 14+
  • Edge 90+ (Chromium-based)

Internet Explorer is not supported due to its lack of modern JavaScript features.

Does it work on mobile devices?

Yes! DevDocs is fully responsive and works well on:

  • Mobile phones (sidebars collapse to overlays)
  • Tablets (adaptive layout based on screen size)
  • Touch interfaces (touch-friendly navigation and interactions)

Common Issues & Troubleshooting

CORS Errors

Problem: Getting "Cross-Origin Request Blocked" errors in the console.

Solution: You're opening HTML files directly in the browser (`file://` protocol). Start an HTTP server instead: 

# Python 3
python -m http.server 8000

# Python 2  
python -m SimpleHTTPServer 8000

# VS Code
Use Live Server extension

# Node.js
npx serve .

Navigation Not Building

Problem: Left sidebar shows "Loading..." indefinitely.

Solutions:

  • Check browser console for JavaScript errors
  • Verify `NAV_DATA` array syntax in `app.js`
  • Ensure all referenced page files exist
  • Clear browser cache and reload

Pages Not Loading

Problem: Clicking navigation items doesn't load content.

Solutions:

  • Check that page paths in `NAV_DATA` are correct
  • Verify files exist in the specified locations
  • Look for 404 errors in browser DevTools Network tab
  • Ensure proper file permissions on your web server

Search Index Errors

Problem: Search returns no results or console shows index errors.

Solutions:

  • Wait for indexing to complete (check console messages)
  • Verify all pages in `NAV_DATA` are accessible
  • Check for malformed HTML that breaks content parsing
  • Reduce page sizes if memory errors occur

Styling Issues

Problem: Layout looks broken or CSS isn't applying.

Solutions:

  • Verify `styles.css` is loading correctly
  • Check for CSS conflicts with custom styles
  • Use browser DevTools to inspect computed styles
  • Clear browser cache to reload stylesheets
