Database Connections

While DevDocs itself requires no database, real-world deployments often integrate with one — to store user preferences, feedback submissions, search analytics, or CMS-managed content. This page covers patterns for connecting to common databases from the backend services that support DevDocs.

ℹ Server-Side Only

Database connections are always server-side. Browser JavaScript cannot connect directly to PostgreSQL, MySQL, or MongoDB. Use a backend API layer (Node.js, Python, etc.) that DevDocs calls over HTTP.

Connection Pooling

Always use a connection pool rather than creating a new connection per request. Pools reuse existing connections, dramatically reducing latency and resource consumption.

PostgreSQL with pg (Node.js)

// db.js
import { Pool } from 'pg';

const pool = new Pool({
  host:     process.env.DB_HOST,
  port:     parseInt(process.env.DB_PORT ?? '5432'),
  database: process.env.DB_NAME,
  user:     process.env.DB_USER,
  password: process.env.DB_PASS,
  max:      10,         // Maximum connections in pool
  idleTimeoutMillis: 30000,
  connectionTimeoutMillis: 2000,
  ssl: process.env.NODE_ENV === 'production'
    ? { rejectUnauthorized: true }
    : false
});

export async function query(sql, params = []) {
  const client = await pool.connect();
  try {
    const result = await client.query(sql, params);
    return result.rows;
  } finally {
    client.release();
  }
}

MySQL / MariaDB

import mysql from 'mysql2/promise';

const pool = mysql.createPool({
  host:            process.env.DB_HOST,
  user:            process.env.DB_USER,
  password:        process.env.DB_PASS,
  database:        process.env.DB_NAME,
  waitForConnections: true,
  connectionLimit:    10,
  queueLimit:         0
});

export async function query(sql, params = []) {
  const [rows] = await pool.execute(sql, params);
  return rows;
}

MongoDB

import { MongoClient } from 'mongodb';

let client;
let db;

export async function getDb() {
  if (!client) {
    client = new MongoClient(process.env.MONGODB_URI);
    await client.connect();
    db = client.db(process.env.DB_NAME);
  }
  return db;
}

// Usage
const db        = await getDb();
const pageViews = await db.collection('analytics').find({ page: '/intro' }).toArray();

Parameterized Queries

Always use parameterized queries to prevent SQL injection. Never concatenate user input directly into SQL strings: 

// SAFE — parameterized
const rows = await query(
  'SELECT * FROM pages WHERE slug = $1 AND published = true',
  [userInput]
);

// DANGEROUS — never do this
const rows = await query(
  `SELECT * FROM pages WHERE slug = '${userInput}'`
);

Schema Migrations

Manage schema changes with a migrations tool to keep development, staging, and production in sync: 

# Using node-pg-migrate
npx node-pg-migrate up

# Create a new migration
npx node-pg-migrate create add-page-views-table
// migrations/1713000000000_add-page-views-table.js
export const up = (pgm) => {
  pgm.createTable('page_views', {
    id:         { type: 'serial', primaryKey: true },
    page_id:    { type: 'varchar(100)', notNull: true },
    viewed_at:  { type: 'timestamptz', default: pgm.func('NOW()') },
    user_agent: { type: 'text' }
  });

  pgm.createIndex('page_views', 'page_id');
};

export const down = (pgm) => {
  pgm.dropTable('page_views');
};

Database Health Check

export async function checkDbHealth() {
  try {
    await query('SELECT 1');
    return { status: 'ok' };
  } catch (err) {
    return { status: 'error', message: err.message };
  }
}

// Expose in health endpoint
app.get('/health', async (req, res) => {
  const db = await checkDbHealth();
  const status = db.status === 'ok' ? 200 : 503;
  res.status(status).json({ db });
});
