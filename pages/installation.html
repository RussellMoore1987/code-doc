Installation

This guide walks you through getting DevDocs running locally on your machine. There is no build step, no package manager, and no framework — just a few files and a local HTTP server.

Prerequisites

Before you begin, make sure you have one of the following available:

  • VS Code with the Live Server extension installed, or
  • Python 3.x — ships with a built-in HTTP server, or
  • Node.js — use the npx serve package if preferred.
ℹ Tip

Any HTTP server will do. The only real requirement is that files are served over http:// or https:// so the Fetch API works correctly. A file:// origin will fail with CORS errors.

Download the Project

Clone this repository or download the ZIP and extract it to a local folder:

# Using git
git clone https://github.com/example/devdocs.git
cd devdocs

# Or download the ZIP and extract:
unzip devdocs.zip
cd devdocs

The project directory should look like this after extraction:

devdocs/
├── index.html
├── styles.css
├── app.js
└── pages/
    ├── intro.html
    ├── installation.html
    ├── first-steps.html
    └── deep-topic.html

Starting a Local Server

Option A — VS Code Live Server

  1. Open the devdocs folder in VS Code.
  2. Install the Live Server extension (ms-vscode.live-server) if you haven't already.
  3. Right-click index.html in the Explorer panel and choose Open with Live Server.
  4. Your browser will open automatically at http://127.0.0.1:5500.

Option B — Python HTTP Server

From inside the project folder, run:

# Python 3
python -m http.server 8080

# Then open in your browser:
# http://localhost:8080

Option C — Node.js serve

npx serve .
# Opens at http://localhost:3000 (or the next available port)

Initial Configuration

Once the server is running and you can see the app in your browser, there are two things you will typically want to customise right away.

Site Title

Open index.html and change the <title> tag and the .brand-title span text to match your project name: 

<title>My Project Docs</title>
...
<span class="brand-title">My Project</span>

Navigation Structure

Open app.js and edit the NAV_DATA array near the top of the file. Each section group has a label and a children array. Each leaf item needs an id, a label, and a page path. 

const NAV_DATA = [
  {
    id: 'my-section',
    label: 'My Section',
    children: [
      { id: 'page-one', label: 'Page One', page: 'pages/page-one.html' },
      { id: 'page-two', label: 'Page Two', page: 'pages/page-two.html' },
    ]
  }
];

Note on nesting depth

The tree renderer supports up to four levels of nesting. Beyond that, indentation will continue to work but may become visually tight at narrower sidebar widths. Three levels is recommended for most projects.

Adding Pages

Each documentation page is a plain HTML file inside the pages/ directory. Pages must:

  1. Be wrapped in a single <div class="content-body"> element.
  2. Use heading tags (h1–h4) with unique id attributes (or let app.js auto-generate them from the heading text).
  3. Contain only the page body — no <html>, <head>, or <body> tags.
<!-- pages/my-page.html -->
<div class="content-body">
  <h1>My Page Title</h1>
  <p>Your content here.</p>
  <h2 id="section-one">Section One</h2>
  <p>More content.</p>
</div>
⚠ Heads up

If you omit id attributes from headings, app.js will auto-generate them from the heading text using a slug algorithm. This is convenient but means IDs can change if you rename headings. Use explicit IDs for any anchor links you share with others.
