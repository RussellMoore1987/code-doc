Common Issues

This page documents the most frequently encountered problems when setting up and running DevDocs, along with step-by-step solutions. If your issue isn't listed here, check the Error Codes reference or the FAQ.

ℹ Quick Check

Most issues have one of three root causes: the site is not being served over HTTP (using file:// instead), a file is missing or misnamed, or a JavaScript error is blocking initialization. Always start by checking the browser console.

Pages Won't Load

CORS / Fetch Error

Symptom: The content area shows "Error loading page" and the console shows a CORS or Failed to fetch error.

Cause: The site is opened as a file:// URL instead of via a local HTTP server.

Solution:

# Option A — Python
cd /path/to/devdocs
python -m http.server 8080
# Open http://localhost:8080

# Option B — Node.js
npx serve .
# Open http://localhost:3000

# Option C — VS Code Live Server
# Right-click index.html → Open with Live Server

Navigation Tree Is Empty

Symptom: The left sidebar is blank or shows no navigation items.

Possible causes and fixes:

  1. NAV_DATA is empty. Check that your entries are correctly formatted objects with at least id and label fields.
  2. JavaScript error halted execution. Open the browser console. If init() threw before buildNavTree() was called, the tree will not render. Fix the underlying error first.
  3. #left-nav element missing. Verify that index.html contains <nav id="left-nav">.

Theme Resets on Every Reload

Symptom: The theme always defaults to light (or dark) even after toggling.

Cause: localStorage is blocked (private browsing mode) or the storage key has been cleared.

Solution:

// Check in console:
console.log(localStorage.getItem('devdocs-theme'));

// If null, manually set it:
localStorage.setItem('devdocs-theme', 'dark');

Search Returns No Results

Symptom: Every search returns "No results found" even for text that visibly appears on pages.

Likely causes:

  • The search index has not finished building. Wait a few seconds after the page loads before searching — buildSearchIndex() runs asynchronously.
  • Pages could not be fetched during index construction. Check the console for Failed to fetch warnings per page.
  • The NAV_DATA entry for the page lacks a page field, so the page is never added to the index.

Sidebar Drag Resize Not Working

Symptom: Dragging the resize handle does not change the sidebar width.

Fix: Check that no CSS pointer-events: none rule is applied to the resize handle or its container. The handle must receive mousedown events.

// Verify in console:
document.getElementById('resize-left').style.pointerEvents

Right Nav Shows "No Sections"

Symptom: The right outline panel shows "No sections on this page" even for pages with many headings.

Cause: The headings in the page fragment are not wrapped in the expected structure, or they are h5/h6 which buildRightNav() does not include.

Fix: Use only h1–h4 tags for section headings that should appear in the outline.

Scroll Spy Highlights Wrong Section

If the right nav highlights the wrong section while scrolling, adjust the rootMargin in setupScrollSpy(): 

const observer = new IntersectionObserver(callback, {
  root:       elContentScroll,
  rootMargin: '0px 0px -70% 0px',  // ← Increase -70% to -80% for tighter detection
  threshold:  0
});
