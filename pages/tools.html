DevDocs Tools

These sections cover the various tools/elements available in the DevDocs.

Citations

Wrap any text in a <span class="citation"> element with a data-citation attribute to turn it into an academic-style citation. On page load the system scans the page, appends a numbered superscript marker (e.g. [1]) right after the wrapped text, and automatically builds a References section at the bottom of the page. No manual numbering or list-keeping required.

Syntax

With a link:

<span
    class="citation"
    data-citation="This feature was introduced in Laravel 11."
    data-link="https://laravel.com/docs/11.x">
    Laravel documentation
</span>

Text only (no link):

<span class="citation" data-citation="Internal design note, 2026.">design note</span>

data-citation is the reference text shown in the References section. data-link is optional; when present it is rendered as a hyperlink that opens in a new tab (target="_blank" with rel="noopener noreferrer").

ℹ Behaviour
  • Citations are numbered sequentially in the order they appear.
  • Identical citations (same text and link) reuse the same number.
  • Clicking a superscript scrolls to its matching reference; each reference has a ↩ Back link that returns to the citation you clicked.
  • The References section is rebuilt on every page load, so it always matches the page.

Doc Link Usage

Use a[data-nav] links anywhere in page content to navigate to another page - or a specific section of another page - without a full page reload. The left sidebar will open the correct menu automatically.

Syntax

Page only:

<a href="#navId" data-nav="navId">Link text</a>

Page + section:

<a href="#navId#sectionId" data-nav="navId" data-section="sectionId">Link text</a>

In-page navigation only:

<a href="#sectionId" data-target="sectionId">Link text</a>

navId is the data-id data attribute of the nav item in the left hand menu, or page name in URL, e.g. image-processing in .../index.html#image-processing.

sectionId is the data-target data attribute of any heading on the target page, right hand menu, or in the URL as e.g. filters-effects in .../index.html#image-processing#filters-effects.

Images

DevDocs includes a few simple ways to display images. Images are supported both as single images and galleries, and includes a built-in lightbox for viewing larger versions of the images.

Single Images

To add a single image, use an <img> tag with the .img-block class. The image will be sized to fit the content area while maintaining its aspect ratio. If the image is smaller than the content width, it will be displayed at its natural size.

Single Image With Lightbox

To add a single image with a lightbox, use an <img> tag with the .img-block and .image-modal classes. Clicking on it will open a larger version in a lightbox. The following images show how the alt text and caption are used in the lightbox:

  • First image: has no alt text, so no text will be used in the lightbox.
  • Second image: has alt text, so the alt text will be used in the lightbox.
  • Third image: is wrapped in a figure with a caption, but has no alt text, so the caption text will be used in the lightbox.
  • Fourth image: is wrapped in a figure with a caption and has alt text, so the alt text will be used in the lightbox and the caption will be ignored in the lightbox.

Tooltip Usage

Add data-tooltip="…" to any element to give it a tooltip. Use the optional data-tooltip-position attribute to set a preferred direction. If the tooltip would overflow the screen, it flips to a better position automatically.

Syntax
<!-- Plain-text tooltip (default position: top) -->
        <button data-tooltip="Tooltip text here">Hover me</button>
    
        <!-- With preferred position -->
        <button data-tooltip="Opens settings" data-tooltip-position="right">⚙</button>
    
        <!-- HTML tooltip - write real HTML inside a <template> element -->
        <template id="my-tip"><strong>Bold</strong> and <em>italic</em><br>Line two</template>
        <button data-tooltip-template="my-tip">Hover me</button>
    
        <!-- Available positions -->
        top | top-left | top-right
        bottom | bottom-left | bottom-right
        left | right
