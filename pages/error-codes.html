Error Codes Reference

This page documents all error codes produced by DevDocs and its associated backend services. Each code includes a description, likely causes, and recommended resolution steps.

ℹ Error Format

Errors are reported in the format DDXXX where DD is the subsystem prefix and XXX is a three-digit code. HTTP status codes follow standard RFC 9110 conventions.

Navigation Errors (DD1xx)

CodeMessageCauseResolution
DD100 Navigation item not found The navId in the URL hash does not exist in NAV_DATA. Check that the id is spelled correctly and exists in NAV_DATA.
DD101 Page load failed The HTML fragment could not be fetched (HTTP 4xx/5xx or network error). Verify the file exists in pages/ and the server is running.
DD102 Re-entrant navigation blocked navigateTo() was called while a navigation was already in progress. Wait for the current navigation to complete before triggering another.
DD103 Section not found on page The sectionId does not match any heading id on the loaded page. Ensure the heading has the expected id attribute.

Search Errors (DD2xx)

CodeMessageCauseResolution
DD200 Search index empty No pages were indexed — all fetch requests failed. Ensure the server is running and all pages in NAV_DATA are accessible.
DD201 Query too long The search query exceeds 25 characters. Shorten the query and try again.
DD202 Too many results The query matches more than 250 sections. Use a more specific search term.
DD203 Index build failed An uncaught exception occurred during index construction. Check the browser console for the underlying error and fix the offending page.

Configuration Errors (DD3xx)

CodeMessageCauseResolution
DD300 Duplicate nav id Two or more items in NAV_DATA share the same id. Ensure every id is globally unique.
DD301 Missing DOM element A required element (e.g., #sidebar-left) is absent from index.html. Restore the element or check for typos in the element's id.
DD302 Invalid theme value An unsupported value was passed to applyTheme(). Use only 'light' or 'dark'.

HTTP Status Codes

When page loading fails, the error message includes the HTTP status from the failed fetch:

StatusMeaningCommon Cause
404Not FoundThe .html file does not exist in the pages/ directory.
403ForbiddenFile permissions or a web server ACL is blocking access.
0 / network errorNo ResponseThe local dev server is not running. Run python -m http.server.
500Server ErrorAn error in a server-side preprocessor or proxy.

Reporting Unknown Errors

If you encounter an error not listed here, gather the following information before reporting:

  1. Full browser console output (copy all errors and warnings).
  2. The URL in the address bar at the time of the error.
  3. Browser name, version, and operating system.
  4. Steps to reproduce the issue reliably.
  5. Whether the issue occurs with all browser extensions disabled.
