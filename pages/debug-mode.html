Debug Mode

Debug mode enables verbose logging, diagnostic overlays, and developer tooling that would be too noisy or risky to expose in production. This page covers how to activate debug mode, what it exposes, and how to use it effectively.

ℹ Development Only

Debug mode is designed for local development and controlled staging environments. Never enable it in production — it exposes internal state and increases console noise which can mask real errors.

Enabling Debug Mode

URL Parameter

Append ?debug=1 to any URL to activate debug mode for that session: 

http://localhost:8080/?debug=1
http://localhost:8080/#intro?debug=1

localStorage Flag

Persist debug mode across page reloads by setting a localStorage flag in the browser console: 

// Enable
localStorage.setItem('devdocs-debug', '1');
location.reload();

// Disable
localStorage.removeItem('devdocs-debug');
location.reload();

Detecting Debug Mode in Code

const DEBUG = new URLSearchParams(window.location.search).get('debug') === '1'
           || localStorage.getItem('devdocs-debug') === '1';

function debugLog(...args) {
  if (DEBUG) console.debug('[DevDocs]', ...args);
}

Diagnostic Overlay

Enable a visual overlay that shows the current route, active nav item, and scroll spy state: 

if (DEBUG) {
  const overlay = document.createElement('div');
  overlay.id = 'debug-overlay';
  overlay.style.cssText = `
    position: fixed; bottom: 8px; left: 8px; z-index: 9999;
    background: rgba(0,0,0,0.8); color: #0f0; font: 12px monospace;
    padding: 8px 12px; border-radius: 4px; max-width: 320px;
    pointer-events: none;
  `;
  document.body.appendChild(overlay);

  // Update on every navigation
  EventBus.on('nav:after', ({ navId, sectionId }) => {
    overlay.innerHTML = `
      navId: ${navId ?? '—'}<br>
      sectionId: ${sectionId ?? '—'}<br>
      leftW: ${state.leftWidth}px | rightW: ${state.rightWidth}px
    `;
  });
}

NAV_DATA Inspector

In debug mode, the entire NAV_DATA structure and runtime state are attached to the global window object for easy inspection in DevTools: 

if (DEBUG) {
  window.__devdocs__ = {
    NAV_DATA,
    state,
    searchState,
    version: '2.4.1'
  };
  console.info('[DevDocs] Debug mode active. Inspect window.__devdocs__');
}

Performance Profiling

function debugMeasure(label, fn) {
  if (!DEBUG) return fn();

  performance.mark(`${label}-start`);
  const result = fn();

  if (result instanceof Promise) {
    return result.finally(() => {
      performance.mark(`${label}-end`);
      performance.measure(label, `${label}-start`, `${label}-end`);
      const entry = performance.getEntriesByName(label).at(-1);
      debugLog(`${label}: ${entry.duration.toFixed(2)}ms`);
    });
  }

  performance.mark(`${label}-end`);
  performance.measure(label, `${label}-start`, `${label}-end`);
  return result;
}
