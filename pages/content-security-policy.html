Content Security Policy

A Content Security Policy (CSP) is an HTTP response header that instructs browsers which sources of scripts, styles, images, and other resources are permitted to load. A properly configured CSP dramatically reduces the risk of Cross-Site Scripting (XSS) attacks.

ℹ Why CSP Matters for DevDocs

DevDocs loads HTML fragments from the pages/ directory dynamically and injects them into the DOM. Without CSP, a malicious page fragment could inject inline scripts and steal user data. CSP prevents this at the browser level.

Recommended CSP Policy

The following policy is a strong starting point for DevDocs:

Content-Security-Policy:
  default-src 'self';
  script-src  'self';
  style-src   'self' 'unsafe-inline';
  img-src     'self' data:;
  font-src    'self';
  connect-src 'self';
  frame-ancestors 'none';
  base-uri    'self';
  form-action 'self';

Directive Explanations

DirectiveValueReason
default-src'self'Fallback for all resource types — only same-origin allowed.
script-src'self'Only app.js from the same origin; no inline scripts, no CDN.
style-src'self' 'unsafe-inline'Inline styles are used by sidebar width calculations; restrict to minimum.
connect-src'self'Restricts fetch() calls to same origin only.
frame-ancestors'none'Prevents the docs from being embedded in iframes (clickjacking defence).

Applying the Policy

Nginx

server {
  add_header Content-Security-Policy
    "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data:; font-src 'self'; connect-src 'self'; frame-ancestors 'none';"
    always;
}

Apache

<IfModule mod_headers.c>
  Header always set Content-Security-Policy \
    "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; frame-ancestors 'none';"
</IfModule>

Meta Tag (development only)

<!-- index.html <head> — use HTTP header in production instead -->
<meta http-equiv="Content-Security-Policy"
      content="default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline';">
⚠ Meta Tag Limitation

The <meta> CSP tag does not support frame-ancestors. Always use the HTTP header in production environments.

Report-Only Mode

Before enforcing a policy, use Content-Security-Policy-Report-Only to audit violations without breaking anything: 

Content-Security-Policy-Report-Only:
  default-src 'self';
  script-src  'self';
  report-uri  /csp-report-endpoint;

Violations are sent as JSON POST requests to report-uri. Log them to identify third-party resources or inline scripts that need to be addressed before switching to enforcement mode.

Eliminating 'unsafe-inline' Styles

The inline style attribute used by the resize feature (el.style.width = n + 'px') prevents dropping 'unsafe-inline' from style-src. Two mitigation options:

  1. Nonce approach: Generate a random nonce per request on the server, inject it into the CSP header and into a <style nonce="..."> block, then move dynamic values to CSS custom properties set via the nonce-protected block.
  2. CSS custom properties: Replace el.style.width with el.style.setProperty('--sidebar-w', n + 'px') and reference var(--sidebar-w) in CSS. This still requires 'unsafe-inline' for inline styles but reduces the scope.
