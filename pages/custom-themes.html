Custom Themes

DevDocs provides a comprehensive theming system that allows complete visual customization through CSS custom properties and modular theme files. This guide covers creating, applying, and managing custom themes for your documentation site.

🎨 Theme Architecture

The theming system uses CSS custom properties (variables) for all colors, spacing, fonts, and animation values. This allows for easy theme switching and maintains consistency across all components without touching core styles.

Theme Structure

Custom themes are defined as CSS files that override the default custom properties. Each theme should define values for all required variables to ensure consistency.

Required Custom Properties

Every custom theme must define these essential properties:

:root[data-theme="custom"] {
  /* Colors */
  --color-bg-primary: #ffffff;
  --color-bg-secondary: #f8f9fa;
  --color-bg-tertiary: #e9ecef;
  --color-text-primary: #212529;
  --color-text-secondary: #6c757d;
  --color-accent: #007bff;
  
  /* Layout */
  --sidebar-width: 280px;
  --content-max-width: 1200px;
  --border-radius: 6px;
  
  /* Typography */
  --font-family-base: 'Inter', -apple-system, sans-serif;
  --font-family-mono: 'Fira Code', 'Monaco', monospace;
  --font-size-base: 16px;
  --line-height-base: 1.6;
}

Theme Variants

Create themed variants by extending base themes with specific modifications: 

/* High contrast theme */
:root[data-theme="high-contrast"] {
  --color-bg-primary: #000000;
  --color-bg-secondary: #1a1a1a;
  --color-text-primary: #ffffff;
  --color-accent: #ff6b35;
  --border-width: 2px;
}

/* Sepia theme */
:root[data-theme="sepia"] {
  --color-bg-primary: #f4f1ea;
  --color-bg-secondary: #e8e2d5;
  --color-text-primary: #5c4b37;
  --color-accent: #8b4513;
}

Creating Custom Themes

Follow these steps to create a completely custom theme that integrates seamlessly with the existing theme switching system.

Theme File Setup

  1. Create a new CSS file in the themes/ directory
  2. Define all required custom properties for your theme
  3. Test with both light and dark mode preferences
  4. Register the theme in the theme switcher configuration
/* themes/corporate.css */
:root[data-theme="corporate"] {
  /* Corporate brand colors */
  --color-bg-primary: #ffffff;
  --color-bg-secondary: #f7f9fc;
  --color-text-primary: #2c3e50;
  --color-accent: #3498db;
  --color-accent-hover: #2980b9;
  
  /* Professional typography */
  --font-family-base: 'Source Sans Pro', sans-serif;
  --font-weight-normal: 400;
  --font-weight-semibold: 600;
  --font-weight-bold: 700;
  
  /* Refined spacing */
  --spacing-xs: 4px;
  --spacing-sm: 8px;
  --spacing-md: 16px;
  --spacing-lg: 24px;
  --spacing-xl: 32px;
}

Advanced Customization

For complex themes, you can override component-specific styles while maintaining the custom property system: 

[data-theme="corporate"] .sidebar-nav {
  border-right: 2px solid var(--color-accent);
}

[data-theme="corporate"] .content-header {
  background: linear-gradient(135deg, 
    var(--color-accent) 0%, 
    var(--color-accent-hover) 100%);
  color: white;
}

[data-theme="corporate"] .callout {
  border-left: 4px solid var(--color-accent);
  background: var(--color-bg-secondary);
}

Theme Management

Manage multiple themes through the theme registry system that handles loading, switching, and persistence.

Theme Registry

Register custom themes in app.js:

const AVAILABLE_THEMES = [
  { id: 'light', name: 'Light', file: null },
  { id: 'dark', name: 'Dark', file: null },
  { id: 'corporate', name: 'Corporate', file: 'themes/corporate.css' },
  { id: 'high-contrast', name: 'High Contrast', file: 'themes/high-contrast.css' },
  { id: 'sepia', name: 'Sepia', file: 'themes/sepia.css' }
];

function loadTheme(themeId) {
  const theme = AVAILABLE_THEMES.find(t => t.id === themeId);
  if (theme && theme.file) {
    const link = document.createElement('link');
    link.rel = 'stylesheet';
    link.href = theme.file;
    document.head.appendChild(link);
  }
  document.documentElement.setAttribute('data-theme', themeId);
}

Dynamic Theme Switching

Enable runtime theme switching with smooth transitions and state persistence: 

function switchTheme(newThemeId) {
  // Add transition class to prevent flash
  document.body.classList.add('theme-transitioning');
  
  // Load new theme
  loadTheme(newThemeId);
  
  // Store preference
  localStorage.setItem('preferred-theme', newThemeId);
  
  // Remove transition class after animation
  setTimeout(() => {
    document.body.classList.remove('theme-transitioning');
  }, 300);
}

Best Practices

  • Test accessibility - Ensure sufficient color contrast ratios
  • Maintain consistency - Use the custom property system exclusively
  • Consider print styles - Define appropriate print-specific overrides
  • Performance impact - Minimize theme file size and complexity
  • User preferences - Respect system dark/light mode preferences
  • Graceful fallbacks - Provide fallback values for all custom properties
💡 Pro Tip

Use CSS custom property cascading to create theme hierarchies. Define base values at the root level and override specific components as needed, allowing for maximum flexibility and maintainability.
