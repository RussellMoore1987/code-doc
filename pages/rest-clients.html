REST Clients

DevDocs can be extended to fetch live data from REST APIs — for dynamic search indexes, user-specific content, analytics dashboards, or CMS-managed pages. This page covers best practices for writing robust REST client code in a static site context.

ℹ The Fetch API

Modern browsers include the fetch() API natively. DevDocs already uses it for page loading. You can use the same patterns for any REST endpoint, with no additional library required.

Basic GET Request

async function fetchData(endpoint) {
  const response = await fetch(`/api/v1/${endpoint}`);

  if (!response.ok) {
    throw new Error(`HTTP ${response.status}: ${response.statusText}`);
  }

  return response.json();
}

// Usage
const data = await fetchData('pages/metadata');

HTTP Methods

POST Request

async function createResource(endpoint, body) {
  const response = await fetch(`/api/v1/${endpoint}`, {
    method:  'POST',
    headers: { 'Content-Type': 'application/json' },
    body:    JSON.stringify(body)
  });

  if (!response.ok) throw new Error(`POST failed: ${response.status}`);
  return response.json();
}

PUT and PATCH

// Full replacement
async function updateResource(endpoint, id, body) {
  return fetch(`/api/v1/${endpoint}/${id}`, {
    method:  'PUT',
    headers: { 'Content-Type': 'application/json' },
    body:    JSON.stringify(body)
  }).then(r => r.json());
}

// Partial update
async function patchResource(endpoint, id, fields) {
  return fetch(`/api/v1/${endpoint}/${id}`, {
    method:  'PATCH',
    headers: { 'Content-Type': 'application/json' },
    body:    JSON.stringify(fields)
  }).then(r => r.json());
}

Error Handling

A clean error-handling wrapper normalizes HTTP and network errors into a consistent shape: 

class ApiError extends Error {
  constructor(status, message, body) {
    super(message);
    this.status = status;
    this.body   = body;
  }
}

async function apiFetch(url, options = {}) {
  let response;
  try {
    response = await fetch(url, options);
  } catch (networkError) {
    throw new ApiError(0, 'Network error', null);
  }

  if (!response.ok) {
    const body = await response.json().catch(() => null);
    throw new ApiError(response.status, response.statusText, body);
  }

  return response.json();
}

// Usage
try {
  const data = await apiFetch('/api/v1/pages');
} catch (err) {
  if (err.status === 404) showNotFound();
  else if (err.status === 401) redirectToLogin();
  else showGenericError(err.message);
}

Handling Pagination

async function fetchAllPages(endpoint) {
  const results = [];
  let cursor    = null;

  do {
    const url  = cursor ? `${endpoint}?cursor=${cursor}` : endpoint;
    const page = await apiFetch(url);

    results.push(...page.items);
    cursor = page.nextCursor ?? null;
  } while (cursor);

  return results;
}

Client-Side Caching

const cache = new Map();
const TTL   = 5 * 60 * 1000; // 5 minutes

async function cachedFetch(url) {
  const cached = cache.get(url);

  if (cached && Date.now() - cached.timestamp < TTL) {
    return cached.data;
  }

  const data = await apiFetch(url);
  cache.set(url, { data, timestamp: Date.now() });
  return data;
}

Request Cancellation

Cancel in-flight requests when the user navigates away or types a new search query: 

let activeController = null;

async function searchApi(query) {
  // Cancel any previous request
  activeController?.abort();
  activeController = new AbortController();

  try {
    return await apiFetch(`/api/v1/search?q=${encodeURIComponent(query)}`, {
      signal: activeController.signal
    });
  } catch (err) {
    if (err.name === 'AbortError') return null; // Silently ignore cancellations
    throw err;
  }
}
ℹ Timeout Pattern

To add a timeout to any fetch request, create an AbortController and call controller.abort() after a setTimeout(). The DevDocs app already uses this pattern for managed event listener cleanup.
