CI/CD Integration

Continuous Integration and Continuous Deployment (CI/CD) pipelines automate testing, building, and deploying DevDocs whenever changes are pushed to your repository. This eliminates manual deployment steps and ensures every commit is validated before it reaches production.

⚙️ Zero Build Step Advantage

DevDocs requires no compilation or bundling. CI pipelines are fast — typically under 60 seconds — because the "build" is simply copying files and running validation checks.

GitHub Actions

Deploy to GitHub Pages

# .github/workflows/deploy.yml
name: Deploy Documentation

on:
  push:
    branches: [main]

permissions:
  contents: read
  pages:    write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: true

jobs:
  deploy:
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Validate HTML
        run: npx html-validate pages/*.html index.html

      - name: Setup Pages
        uses: actions/configure-pages@v4

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: '.'

      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4

Build and Push Docker Image

# .github/workflows/docker.yml
name: Build and Push Docker Image

on:
  push:
    tags: ['v*']

jobs:
  docker:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Log in to registry
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          push: true
          tags: |
            ghcr.io/${{ github.repository }}:latest
            ghcr.io/${{ github.repository }}:${{ github.ref_name }}

GitLab CI/CD

# .gitlab-ci.yml
stages:
  - validate
  - deploy

validate:html:
  stage: validate
  image: node:20-alpine
  script:
    - npm install -g html-validate
    - html-validate pages/*.html index.html

pages:
  stage: deploy
  script:
    - mkdir -p public
    - cp -r . public/
  artifacts:
    paths:
      - public
  only:
    - main

Automated Link Checking

Add a broken-link check to catch navigation entries that reference non-existent pages: 

# Install lychee (fast link checker)
cargo install lychee

# Check all links (local + external)
lychee --offline pages/*.html index.html

As a CI step:

- name: Check internal links
  run: |
    npx linkinator . --recurse --skip "https://|http://" --verbosity error

Deployment Strategies

Blue-Green Deployment

Maintain two identical production environments (blue and green). Deploy new versions to the idle environment, run smoke tests, then switch traffic instantly.

Rolling Deployment

Gradually replace old instances with new ones. Works well behind a load balancer with health checks enabled.

Canary Release

Route a small percentage of traffic (e.g., 5%) to the new version, monitor error rates, then gradually increase the traffic split.

Branch Preview Deployments

Automatically deploy a preview of each pull request for stakeholder review: 

- name: Deploy preview
  uses: cloudflare/pages-action@v1
  with:
    apiToken: ${{ secrets.CF_API_TOKEN }}
    accountId: ${{ secrets.CF_ACCOUNT_ID }}
    projectName: devdocs
    directory: .
    gitHubToken: ${{ secrets.GITHUB_TOKEN }}
