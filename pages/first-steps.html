First Steps

With DevDocs installed and your local server running, this guide walks you through the most common day-to-day tasks: writing content, organising navigation, using callout styles, and embedding code samples.

Writing Content

Documentation pages are standard HTML wrapped in a <div class="content-body">. You can use any HTML you like inside — the stylesheet provides sensible defaults for all common elements.

Headings

Use h1 once per page as the page title. Use h2–h4 for sections and sub-sections. Headings are automatically picked up by the right-hand outline and scroll-spy system:

  • h1 — Page title (one per page)
  • h2 — Major section
  • h3 — Sub-section
  • h4 — Minor sub-section (displayed in uppercase with letter-spacing in the default style)
ℹ Best Practice

Always give headings unique, descriptive id attributes. This ensures stable deep-link URLs and predictable scroll-spy behaviour.

Paragraphs & Lists

Standard <p>, <ul>, and <ol> elements all receive default spacing and support nesting:

  1. Ordered lists are great for sequential steps.
  2. Numbers are automatic — just add <li> items.
    • Nested <ul> elements indent automatically.
    • You can mix ordered and unordered at any level.
  3. Keep individual list items concise — one idea per bullet.

Code Blocks

Inline Code

Wrap short identifiers, filenames, and command fragments in <code> tags: const x = 42, index.html, npm install.

Fenced Code Blocks

Use a <pre><code> pair for multi-line samples. The default style provides a subtle background, rounded corners, and horizontal scroll on overflow:

// JavaScript example
function greet(name) {
  if (!name) throw new Error('Name is required');
  return `Hello, ${name}!`;
}

const message = greet('DevDocs');
console.log(message); // "Hello, DevDocs!"
# Shell / Bash example
cd my-project
python -m http.server 8080 &
echo "Server started on port 8080"
open http://localhost:8080

Syntax colouring

The default stylesheet does not include a syntax highlighter to keep dependencies at zero. If you need syntax colouring, drop in highlight.js or Prism.js — both work without a build step.

Callout Boxes

Three callout variants are provided out of the box. Add the callout class plus an optional modifier to a <div>, then include a .callout-title element inside.

Info (default)

ℹ Info

Use this for general notes and supplementary information that readers may find helpful but can skip without missing critical steps.

<div class="callout">
  <div class="callout-title">ℹ Info</div>
  <p>Your note here.</p>
</div>

Warning

⚠ Warning

Use this for actions that could have unintended consequences or commonly trip people up. Don't overuse warnings — readers learn to ignore them if they appear too often.

Danger

✖ Danger

Use this sparingly for irreversible or destructive actions, such as deleting data or overwriting configuration without a backup.

Images & Figures

Wrap images in a <figure class="img-block"> to get the default border, rounded corners, and caption style. Use <figcaption> for an optional caption. 

<figure class="img-block">
  <img src="images/screenshot.png" alt="Application screenshot" />
  <figcaption>The three-column layout at 1440 px wide.</figcaption>
</figure>

Sidebar Layout & Resizing

Both sidebars can be collapsed and resized interactively at any time:

  • Collapse — click the arrow icon (‹ / ›) in the sidebar header.
  • Expand — click the arrow icon in the narrow strip that appears when collapsed.
  • Resize — drag the 5 px divider between the sidebar and the center content area. Widths are persisted to localStorage so your preference is remembered across page loads.

Minimum Widths

The left and right sidebars each enforce a minimum width of 220 px. If you drag a sidebar below that threshold it will spring back to the minimum. You can adjust the MIN_SIDEBAR_WIDTH constant in app.js.

Responsive Layout

DevDocs automatically collapses sidebars on narrow viewports to protect the center content area:

  • Below 1200 px — the right sidebar collapses automatically.
  • Below 800 px — the left sidebar also collapses.

On touch devices you can still expand either sidebar manually using the strip expand button. The responsive collapse only triggers once on load; after that, user preference takes over.

Customising breakpoints

Edit the checkResponsiveCollapse() function in app.js and the matching media queries in styles.css to suit your content.

Summary

You now know how to:

  • Write and structure documentation pages
  • Use inline code, code blocks, and callout boxes
  • Include images with captions
  • Navigate and resize the sidebar layout
