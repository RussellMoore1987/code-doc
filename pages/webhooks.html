Webhooks

Webhooks let external services push real-time notifications to your systems when events occur — a document is published, a CI build completes, or a user submits feedback. This page describes how to configure, receive, and secure webhook integrations within the DevDocs ecosystem.

ℹ Push vs. Pull

Traditional integrations poll an API on a schedule. Webhooks invert this: the sender pushes a payload to your endpoint the moment something happens. This is more efficient and real-time but requires you to expose a public HTTP endpoint.

Payload Format

DevDocs webhook payloads are JSON objects sent via HTTP POST with a Content-Type: application/json header: 

{
  "event":     "page.published",
  "timestamp": "2026-04-22T14:30:00Z",
  "version":   "1",
  "data": {
    "pageId":  "installation",
    "pageUrl": "pages/installation.html",
    "author":  "alice@example.com",
    "title":   "Installation Guide"
  }
}

Event Types

EventTrigger
page.publishedA page is published or updated in production
page.deletedA page is removed from the navigation
deploy.successA deployment completes successfully
deploy.failureA deployment fails
feedback.submittedA user submits feedback on a page

Receiving Webhooks

Your endpoint must respond with HTTP 200 within 10 seconds to acknowledge receipt:

// Node.js / Express example
import express from 'express';
import crypto  from 'crypto';

const app = express();
app.use(express.json());

app.post('/webhooks/devdocs', (req, res) => {
  // Immediately acknowledge
  res.sendStatus(200);

  // Process asynchronously
  processWebhookAsync(req.body).catch(console.error);
});

async function processWebhookAsync(payload) {
  const { event, data } = payload;

  if (event === 'page.published') {
    await invalidateCacheForPage(data.pageUrl);
    await notifySlack(`📄 Page published: ${data.title}`);
  }
}

Securing Webhooks

Verify that incoming requests originate from the expected sender by validating an HMAC signature in the request header: 

const WEBHOOK_SECRET = process.env.WEBHOOK_SECRET;

function verifySignature(req) {
  const signature = req.headers['x-devdocs-signature-256'];
  if (!signature) return false;

  const expected = 'sha256=' + crypto
    .createHmac('sha256', WEBHOOK_SECRET)
    .update(JSON.stringify(req.body))
    .digest('hex');

  // Use timingSafeEqual to prevent timing attacks
  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expected)
  );
}

app.post('/webhooks/devdocs', (req, res) => {
  if (!verifySignature(req)) {
    return res.status(401).send('Invalid signature');
  }
  res.sendStatus(200);
  processWebhookAsync(req.body).catch(console.error);
});

Retry Policy

If your endpoint returns a non-2xx status or times out, the sender will retry with exponential backoff:

AttemptDelay
1st retry30 seconds
2nd retry5 minutes
3rd retry30 minutes
4th retry2 hours

Make your webhook handler idempotent — processing the same event twice should produce the same result. Use the timestamp + event combination as a deduplication key.
