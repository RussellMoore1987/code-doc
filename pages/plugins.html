Plugins

DevDocs supports extensibility through a flexible plugin system that allows you to add custom functionality without modifying core files. This guide covers creating, installing, and managing plugins to enhance your documentation experience.

🔌 Plugin Philosophy

Plugins in DevDocs are lightweight JavaScript modules that hook into the application lifecycle. They can extend search capabilities, add content transformations, integrate external services, or customize the UI without breaking core functionality.

Plugin Architecture

DevDocs plugins follow a simple event-driven architecture. Plugins register themselves with the main application and can listen for specific lifecycle events or extend existing functionality.

Plugin Lifecycle Events

Plugins can hook into these application events:

  • app:init - Application initialization complete
  • nav:built - Navigation tree has been rendered
  • page:loaded - New page content loaded via Ajax
  • search:indexed - Search index building complete
  • theme:changed - User toggled light/dark theme
  • sidebar:resized - Left or right sidebar width changed

Core Plugin Interface

All plugins implement a standard interface for consistency:

class DevDocsPlugin {
  constructor(options = {}) {
    this.name = 'My Plugin';
    this.version = '1.0.0';
    this.options = options;
  }

  // Called when plugin is registered
  init(app) {
    this.app = app;
    this.setupEventListeners();
  }

  // Plugin-specific setup
  setupEventListeners() {
    this.app.on('page:loaded', this.onPageLoaded.bind(this));
  }

  // Handle page load events
  onPageLoaded(pageData) {
    // Your custom logic here
  }

  // Cleanup when plugin is disabled
  destroy() {
    // Remove listeners, clean up DOM, etc.
  }
}

Built-in Plugins

DevDocs includes several built-in plugins that demonstrate the plugin architecture and provide commonly requested functionality out of the box.

Code Copy Plugin

Automatically adds copy-to-clipboard buttons to all code blocks. Users can click to copy code snippets without manual text selection. 

// Enable in app.js
plugins: [
  new CodeCopyPlugin({
    buttonText: 'Copy',
    successText: 'Copied!',
    position: 'top-right'
  })
]

External Links Plugin

Detects external links in content and adds appropriate security attributes and visual indicators. Helps users understand when they're leaving the documentation site.

  • Adds rel="noopener noreferrer" to external links
  • Opens external links in new tabs
  • Adds visual icon to indicate external destination
  • Excludes localhost and development URLs

Analytics Plugin

Integrates with popular analytics services to track page views, search queries, and user interactions without compromising privacy or performance. 

// Google Analytics integration
new AnalyticsPlugin({
  provider: 'ga4',
  trackingId: 'G-XXXXXXXXXX',
  trackPageViews: true,
  trackSearchQueries: false,  // Respect user privacy
  trackScrollDepth: true
})

Creating Custom Plugins

Building custom plugins lets you tailor DevDocs to your specific documentation needs. Here's a step-by-step guide to creating your first plugin.

Simple Plugin Example

Let's create a plugin that adds reading time estimates to pages:

class ReadingTimePlugin extends DevDocsPlugin {
  constructor(options = {}) {
    super();
    this.name = 'Reading Time';
    this.wordsPerMinute = options.wordsPerMinute || 200;
  }

  init(app) {
    super.init(app);
    this.app.on('page:loaded', this.addReadingTime.bind(this));
  }

  addReadingTime(pageData) {
    const content = document.querySelector('.content-body');
    if (!content) return;

    // Count words in visible text
    const text = content.textContent || content.innerText || '';
    const wordCount = text.trim().split(/\s+/).length;
    const readingTime = Math.ceil(wordCount / this.wordsPerMinute);

    // Add reading time indicator
    const timeIndicator = document.createElement('div');
    timeIndicator.className = 'reading-time';
    timeIndicator.innerHTML = `📖 ${readingTime} min read`;
    
    const title = content.querySelector('h1');
    if (title) {
      title.insertAdjacentElement('afterend', timeIndicator);
    }
  }
}

Advanced Plugin Features

More sophisticated plugins can integrate deeply with DevDocs functionality:

  • Search Enhancement - Add custom search filters or result ranking
  • Content Transformation - Process Markdown, syntax highlighting, diagrams
  • UI Extensions - Add toolbar buttons, status indicators, floating panels
  • Data Integration - Pull content from CMSs, APIs, or external documentation
🛠️ Development Tip

Use browser DevTools to test plugins during development. The plugin system includes debug logging that shows registration timing, event firing, and performance metrics.

Plugin Management

Managing multiple plugins requires careful attention to loading order, dependencies, and potential conflicts between plugins.

Plugin Registration

Register plugins in app.js after the core application initializes:

// app.js - Plugin registration
const plugins = [
  new CodeCopyPlugin(),
  new ReadingTimePlugin({ wordsPerMinute: 180 }),
  new ExternalLinksPlugin({ openInNewTab: true }),
  new AnalyticsPlugin({ 
    provider: 'plausible',
    domain: 'docs.example.com'
  })
];

// Initialize plugins after DOM is ready
document.addEventListener('DOMContentLoaded', () => {
  initializeApp();
  
  plugins.forEach(plugin => {
    try {
      plugin.init(app);
      console.log(`✅ Plugin "${plugin.name}" loaded successfully`);
    } catch (error) {
      console.error(`❌ Plugin "${plugin.name}" failed to load:`, error);
    }
  });
});

Plugin Configuration

Plugins should accept configuration objects to make them reusable across different documentation sites. Store plugin settings in a separate configuration file: 

// plugins-config.js
export const pluginConfig = {
  codeCopy: {
    buttonText: 'Copy Code',
    successText: '✅ Copied!',
    showLanguage: true,
    timeout: 2000
  },
  
  readingTime: {
    wordsPerMinute: 200,
    showIcon: true,
    position: 'after-title'  
  },
  
  analytics: {
    provider: 'none',  // Disable by default
    respectDoNotTrack: true,
    anonymizeIp: true
  }
};

Development Guidelines

When creating plugins for the DevDocs ecosystem, follow these best practices:

  • Keep plugins focused on a single responsibility
  • Provide comprehensive configuration options
  • Include error handling and graceful degradation
  • Document plugin APIs and usage examples
  • Test across different browsers and screen sizes
  • Consider accessibility implications of UI changes
