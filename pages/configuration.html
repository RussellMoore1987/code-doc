Configuration

DevDocs offers extensive configuration options to customize appearance, behavior, and functionality. This comprehensive guide covers all configuration aspects, from basic theming to advanced customization techniques.

⚙️ Configuration Philosophy

DevDocs follows a "configuration over convention" approach for customization while maintaining sensible defaults. Most changes require only CSS custom property overrides or JavaScript configuration object modifications.

Basic Configuration

Start with these fundamental configuration options to personalize your DevDocs installation without modifying core files. **Search Test***

Site Identity

Configure basic site information in index.html: 

<!-- Update page title and branding -->
<title>YourProject Docs</title>

<div class="header-brand">
  <img src="/images/your-logo.svg" alt="Your Logo" class="logo">
  <span class="brand-title">YourProject</span>
</div>

<!-- Update footer information -->
<span class="footer-copy">&copy; 2026 YourProject. All rights reserved.</span>

Theming Configuration

DevDocs uses CSS custom properties for comprehensive theming support. All visual aspects can be customized without touching the core stylesheet.

Color Scheme Customization

Override color tokens to match your brand: 

/* Custom theme - add to index.html or separate CSS file */
:root {
  /* Primary colors */
  --accent: #2563eb;           /* Blue primary */
  --accent-light: #dbeafe;     /* Light blue background */
  
  /* Brand colors */
  --bg-header: #1f2937;        /* Dark header */
  --text-brand: #ffffff;       /* White brand text */
  
  /* Navigation */
  --nav-active-bar: #2563eb;   /* Active page indicator */
  --nav-hover: #f3f4f6;        /* Navigation hover state */
  
  /* Callouts */
  --callout-border: #2563eb;   /* Callout border color */
  --bg-callout: #eff6ff;       /* Callout background */
}

/* Dark theme overrides */
[data-theme="dark"] {
  --accent: #60a5fa;
  --accent-light: #1e3a8a;
  --bg-header: #111827;
  --nav-hover: #374151;
  --bg-callout: #1e3a8a;

Search Configuration **Search Test***

Customize search behavior and indexing: 

// Search configuration object
const SEARCH_CONFIG = {
  // Indexing options
  maxIndexSize: 1000,           // Maximum number of indexed items
  minWordLength: 3,             // Minimum word length for indexing

  // Performance settings

Plugin Configuration

Configure and manage plugins through a centralized configuration system.

Plugin Registry

Register and configure plugins in a dedicated configuration file: 

// plugins.config.js
export const PLUGIN_CONFIG = {
  // Code copy functionality
  codeCopy: {
    enabled: true,
    buttonText: 'Copy',
    successText: 'Copied!',
    errorText: 'Failed to copy',
    timeout: 2000,
    position: 'top-right'
  },
  
  // Reading time estimates
  readingTime: {
    enabled: true,
    wordsPerMinute: 200,
    showIcon: true,
    template: '📖 ${time} min read'
  },
  
  // External link handling
  externalLinks: {
    enabled: true,
    openInNewTab: true,
    showIcon: true,
    excludeDomains: ['localhost', '127.0.0.1']
  },
  
  // Analytics integration
  analytics: {
    enabled: false,  // Disabled by default
    provider: 'none',
    trackingId: null,
    anonymizeIp: true,
    respectDoNotTrack: true
  }
};

Conditional Plugin Loading

Load plugins conditionally based on environment or user preferences: 

// Conditional plugin loading
const plugins = [];

// Always load core functionality plugins
plugins.push(new CodeCopyPlugin(PLUGIN_CONFIG.codeCopy));
plugins.push(new ExternalLinksPlugin(PLUGIN_CONFIG.externalLinks));

// Load analytics only in production
if (location.hostname !== 'localhost' && PLUGIN_CONFIG.analytics.enabled) {
  plugins.push(new AnalyticsPlugin(PLUGIN_CONFIG.analytics));
}

// Load development plugins in dev mode
if (location.search.includes('debug=true')) {
  plugins.push(new DebugPlugin());
  plugins.push(new PerformanceMonitorPlugin());
}

Environment Configuration

Configure DevDocs for different deployment environments and use cases.

Deployment Environments

Set up environment-specific configurations: 

// config/environments.js
const ENVIRONMENTS = {
  development: {
    debug: true,
    analytics: false,
    caching: false,
    hotReload: true,
    performanceMonitoring: true
  },
  
  staging: {
    debug: false,
    analytics: false,
    caching: true,
    hotReload: false,
    performanceMonitoring: true
  },
  
  production: {
    debug: false,
    analytics: true,
    caching: true,
    hotReload: false,
    performanceMonitoring: false
  }
};

// Auto-detect environment
const currentEnv = location.hostname === 'localhost' ? 'development' :
                  location.hostname.includes('staging') ? 'staging' : 'production';

export const ENV_CONFIG = ENVIRONMENTS[currentEnv];

CORS Configuration

Configure CORS headers for Ajax content loading: 

# .htaccess example
<IfModule mod_headers.c>
  # Allow same-origin requests
  Header always set Access-Control-Allow-Origin "*"
  Header always set Access-Control-Allow-Methods "GET, OPTIONS"
  Header always set Access-Control-Allow-Headers "Content-Type"
  
  # Security headers
  Header always set X-Content-Type-Options nosniff
  Header always set X-Frame-Options DENY
  Header always set X-XSS-Protection "1; mode=block"
</IfModule>
🔒 Security Best Practices

Always validate configuration input, sanitize dynamic content, and regularly update dependencies. Consider implementing subresource integrity (SRI) for any external resources you may add to DevDocs.
