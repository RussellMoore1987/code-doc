Support & Resources

Whether you are stuck on a configuration problem, have found a bug, or want to contribute to DevDocs, this page points you to the right resources and explains how to get help efficiently.

ℹ Before You Ask

Most questions are answered in the documentation. Use the search function (Ctrl/Cmd + K) to search across all pages before opening a support ticket. Also check the FAQ and Common Issues pages.

Getting Help

GitHub Discussions

For questions, ideas, and general discussions about DevDocs, use GitHub Discussions on the project repository. This is the best place for questions that don't qualify as bug reports.

  • Search existing discussions before opening a new one.
  • Include your DevDocs version, browser, and OS in your post.
  • Share a minimal reproducible example where possible.

Bug Reports

File bug reports as GitHub Issues. A good bug report includes:

  1. A clear, descriptive title.
  2. Steps to reproduce the bug (numbered, precise).
  3. What you expected to happen.
  4. What actually happened (including console errors).
  5. Browser DevTools console output (screenshot or text).
  6. Your environment: DevDocs version, browser + version, OS.

Feature Requests

Feature requests are welcome. Use the GitHub Issues tracker with the enhancement label. Describe:

  • The problem your feature request solves.
  • Your proposed solution (if any).
  • Alternatives you have considered.
  • How important this is to your use case (nice-to-have vs. blocking).

Contributing

Code Contributions

  1. Fork the repository and create a feature branch: git checkout -b feat/my-feature
  2. Make your changes with clear, focused commits.
  3. Test in multiple browsers (Chrome, Firefox, Safari, Edge).
  4. Ensure no new console errors are introduced.
  5. Open a pull request with a description of the change and why it's needed.

Documentation Contributions

To improve or add documentation pages:

  1. Create a new .html fragment in the pages/ directory.
  2. Follow the existing page structure (see Architecture Overview).
  3. Add the corresponding entry to NAV_DATA in app.js.
  4. Test that the page loads and the right nav generates correctly.

Release Notes & Changelog

Every release is documented in the repository's CHANGELOG.md. The changelog follows the Keep a Changelog format with sections for Added, Changed, Deprecated, Removed, Fixed, and Security.
