CLI Reference

DevDocs includes command-line utilities to streamline development, content management, and deployment workflows. This reference covers all available CLI tools and their usage patterns.

🛠️ CLI Tools

DevDocs CLI tools are designed to work with any environment that supports Node.js or Python. Most commands can be run directly from your terminal without additional setup beyond the prerequisites listed in each section.

DevDocs CLI

The main DevDocs CLI provides commands for project initialization, content management, and build processes.

Installation

Install the DevDocs CLI globally via npm:

# Install globally
npm install -g @devdocs/cli

# Or use with npx (no global install)
npx @devdocs/cli --help

# Verify installation
devdocs --version

Project Commands

init

Initialize a new DevDocs project in the current directory:

devdocs init [project-name] [options]

# Examples
devdocs init my-docs
devdocs init --template minimal
devdocs init --no-git --theme dark
Option Description Default
--template <name> Project template (default, minimal, api) default
--theme <name> Initial theme (light, dark, auto) light
--no-git Skip git repository initialization false
--force Overwrite existing files false

serve

Start a local development server with live reload:

devdocs serve [options]

# Examples  
devdocs serve
devdocs serve --port 8080 --open
devdocs serve --host 0.0.0.0 --cors
Option Description Default
--port <number> Server port 3000
--host <address> Server host address localhost
--open Open browser automatically false
--cors Enable CORS headers false
--watch Enable file watching and live reload true

build

Build the project for production deployment:

devdocs build [options]

# Examples
devdocs build
devdocs build --output dist --minify
devdocs build --no-search-index
Option Description Default
--output <dir> Build output directory build
--minify Minify CSS and JavaScript false
--no-search-index Skip search index generation false
--base-url <url> Base URL for deployment /

Content Management

CLI commands for creating, organizing, and validating documentation content.

Page Commands

new-page

Create a new documentation page with proper structure:

devdocs new-page <name> [options]

# Examples
devdocs new-page "Getting Started" --section intro
devdocs new-page "API Guide" --template api --section reference
Option Description Default
--section <name> Navigation section to add page to none
--template <name> Page template (basic, api, tutorial) basic
--slug <name> Custom URL slug auto-generated

validate

Validate content structure and check for issues:

devdocs validate [path] [options]

# Examples
devdocs validate
devdocs validate pages/ --fix-links
devdocs validate --strict --report json
Option Description Default
--fix-links Automatically fix broken internal links false
--strict Enable strict validation (fail on warnings) false
--report <format> Report format (console, json, html) console

generate-nav

Auto-generate navigation structure from file system:

devdocs generate-nav [options]

# Examples
devdocs generate-nav --depth 3
devdocs generate-nav --exclude "*.draft.html" --sort alpha
Option Description Default
--depth <number> Maximum navigation depth 4
--exclude <pattern> Glob pattern for files to exclude none
--sort <method> Sort method (alpha, date, custom) alpha

Search Management

Commands for managing the search index and search functionality.

index

Build or rebuild the search index:

devdocs index [options]

# Examples
devdocs index
devdocs index --verbose --output search-index.json
devdocs index --exclude "pages/archive/**"
Option Description Default
--output <file> Index output file (JSON) auto
--exclude <pattern> Exclude files matching pattern none
--verbose Show detailed indexing progress false
--stemming Enable word stemming false

search-test

Test search functionality with sample queries:

devdocs search-test <query> [options]

# Examples
devdocs search-test "configuration"
devdocs search-test "api reference" --max-results 5 --debug

Theme Management

Commands for creating and managing custom themes.

theme-create

Create a new custom theme:

devdocs theme-create <name> [options]

# Examples  
devdocs theme-create corporate --base light
devdocs theme-create dark-purple --colors purple.json

theme-build

Build and validate a custom theme:

devdocs theme-build <theme-dir> [options]

# Examples
devdocs theme-build themes/corporate --output themes/corporate.css
devdocs theme-build themes/custom --validate --minify

Deployment Commands

Commands for deploying DevDocs to various hosting platforms.

deploy

Deploy to supported hosting platforms:

devdocs deploy <platform> [options]

# Examples
devdocs deploy github-pages --branch gh-pages
devdocs deploy netlify --site-id abc123
devdocs deploy s3 --bucket my-docs-bucket
Platform Required Options Description
github-pages --branch Deploy to GitHub Pages
netlify --site-id Deploy to Netlify
s3 --bucket Deploy to AWS S3
ftp --host, --user Deploy via FTP

Utility Commands

Miscellaneous utility commands for development and maintenance.

convert

Convert content from other documentation formats:

devdocs convert <format> <input> [options]

# Examples
devdocs convert markdown docs/ --output pages/
devdocs convert gitbook book.json --preserve-structure

analyze

Analyze project structure and provide optimization suggestions:

devdocs analyze [options]

# Examples
devdocs analyze --performance --accessibility
devdocs analyze --report html --output analysis.html

clean

Clean up generated files and caches:

devdocs clean [options]

# Examples
devdocs clean
devdocs clean --cache-only

Scripting & Automation

Examples of using DevDocs CLI in scripts and automated workflows.

NPM Scripts Integration

// package.json
{
  "scripts": {
    "docs:dev": "devdocs serve --open",
    "docs:build": "devdocs build --minify",
    "docs:validate": "devdocs validate --strict",
    "docs:deploy": "npm run docs:build && devdocs deploy netlify"
  }
}

CI/CD Examples

# GitHub Actions example (.github/workflows/docs.yml)
name: Deploy Documentation
on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
      - run: npm install -g @devdocs/cli
      - run: devdocs validate --strict
      - run: devdocs build --minify
      - run: devdocs deploy github-pages --branch gh-pages

Batch Operations

# Bash script for content migration
#!/bin/bash

# Convert all markdown files
for file in old-docs/*.md; do
  devdocs convert markdown "$file" --output "pages/$(basename "$file" .md).html"
done

# Rebuild navigation and search
devdocs generate-nav --sort alpha
devdocs index --verbose

# Validate everything
devdocs validate --fix-links
🚀 Pro Tips

Use devdocs --help or devdocs <command> --help for detailed information about any command. Most commands support --dry-run to preview changes before executing them.
