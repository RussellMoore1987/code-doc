Logging Guide

Good logging is the foundation of observable systems. This guide covers what to log, how to structure log output, and how to ship logs from DevDocs deployments to your preferred observability platform.

ℹ Client vs. Server Logging

Browser-side JavaScript can only log to the DevTools console. Structured, persistent logs require a backend service. This guide covers both layers.

Log Levels

Use consistent log levels to allow filtering in production:

Levelconsole MethodWhen to Use
DEBUGconsole.debug()Verbose diagnostic data — dev only
INFOconsole.info()Normal lifecycle events (page loaded, theme changed)
WARNconsole.warn()Recoverable issues (failed to index one page)
ERRORconsole.error()Unexpected failures that affect functionality
 
// Structured client-side logger
const logger = {
  debug: (...args) => DEBUG && console.debug('[DD]', ...args),
  info:  (...args) =>           console.info ('[DD]', ...args),
  warn:  (...args) =>           console.warn ('[DD]', ...args),
  error: (...args) =>           console.error('[DD]', ...args),
};

What to Log

Navigation Events

// Log every page navigation
EventBus.on('nav:page-loaded', ({ url, navId }) => {
  logger.info('Page loaded', { url, navId, timestamp: Date.now() });
});

Errors

// Catch unhandled errors
window.addEventListener('error', (event) => {
  logger.error('Unhandled error', {
    message:  event.message,
    filename: event.filename,
    line:     event.lineno,
    column:   event.colno,
    stack:    event.error?.stack
  });
});

// Catch unhandled promise rejections
window.addEventListener('unhandledrejection', (event) => {
  logger.error('Unhandled promise rejection', {
    reason: event.reason?.message ?? String(event.reason)
  });
});

Structured Logging (Server-Side)

On the server, always log JSON rather than plain strings. Structured logs are indexable, filterable, and machine-readable. 

// Using pino (fast Node.js logger)
import pino from 'pino';

const log = pino({
  level: process.env.LOG_LEVEL ?? 'info',
  formatters: {
    level: (label) => ({ level: label })
  }
});

// Usage
log.info({ pageId: 'intro', duration: 45 }, 'Page served');
log.warn({ pageId: 'missing', status: 404 }, 'Page not found');
log.error({ err }, 'Unhandled error in middleware');

Shipping Logs to a Platform

Datadog

# docker-compose.yml — Datadog log collection
services:
  docs:
    image: devdocs:latest
    labels:
      com.datadoghq.ad.logs: '[{"source":"nginx","service":"devdocs"}]'
    logging:
      driver: json-file

AWS CloudWatch

# ECS task definition log configuration
"logConfiguration": {
  "logDriver": "awslogs",
  "options": {
    "awslogs-group":         "/ecs/devdocs",
    "awslogs-region":        "us-east-1",
    "awslogs-stream-prefix": "ecs"
  }
}

Log Retention Policies

Log TypeRecommended RetentionReason
Access logs90 daysTraffic analysis, incident forensics
Application errors30 daysBug investigation
Security/audit logs1 year+Compliance requirements
Debug logs7 daysHigh volume, low long-term value
⚠ PII in Logs

Never log personally identifiable information (email addresses, IP addresses in some jurisdictions, user IDs linked to individuals) unless required and consented to. Use anonymization or pseudonymization techniques where logging user data is necessary.
