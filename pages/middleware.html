Middleware Pipeline

The middleware pipeline intercepts key operations — page loads, navigation requests, and search queries — allowing plugins and custom code to transform inputs, inject side effects, or abort processing before the core logic executes.

⚙️ Inspired by Express.js

If you have used Express or Koa middleware, the pattern will feel familiar. Each middleware receives a context object and a next function. Call next() to pass control to the next middleware in the chain.

How the Pipeline Works

Middleware functions are executed in registration order. Each function can mutate the context, perform async work, or halt the chain by not calling next(). 

// Middleware signature
async function myMiddleware(ctx, next) {
  // Before — runs before the core handler
  console.log('Before:', ctx.navId);

  await next(); // Pass control forward

  // After — runs after the core handler returns
  console.log('After:', ctx.navId);
}

// Register
Middleware.use('navigation', myMiddleware);

Available Pipelines

Navigation Pipeline

Wraps every call to navigateTo(). The context object contains: 

ctx = {
  navId:       string | null,
  sectionId:   string | null,
  pushHistory: boolean,
  smooth:      boolean,
  abort:       () => void   // Call to cancel navigation entirely
}

Page Load Pipeline

Wraps each fetch() call for page HTML. Useful for caching or request interception: 

ctx = {
  url:      string,
  response: Response | null,  // populated after fetch
  html:     string | null,    // populated after text() parsing
  abort:    () => void
}

Search Pipeline

Wraps performSearch(). Intercept or modify queries before they run:

ctx = {
  query:   string,  // mutable — you may rewrite the query
  results: SearchResult[] | null  // populated after core search
}

Practical Examples

Authentication Guard

Middleware.use('navigation', async (ctx, next) => {
  const protectedIds = ['admin-panel', 'billing'];

  if (protectedIds.includes(ctx.navId) && !isAuthenticated()) {
    ctx.abort();
    redirectToLogin();
    return;
  }

  await next();
});

Simple Page Cache

const cache = new Map();

Middleware.use('page-load', async (ctx, next) => {
  if (cache.has(ctx.url)) {
    ctx.html = cache.get(ctx.url);
    return; // Skip fetch entirely
  }

  await next();

  if (ctx.html) {
    cache.set(ctx.url, ctx.html);
  }
});

Search Query Expansion

Middleware.use('search', async (ctx, next) => {
  // Expand abbreviations before searching
  const expansions = { 'cfg': 'configuration', 'env': 'environment' };
  ctx.query = expansions[ctx.query] ?? ctx.query;
  await next();
});

Middleware Ordering

Middleware executes in the order it is registered. Order matters — an authentication guard must be registered before a logging middleware if you want to avoid logging blocked requests. 

// Auth runs first
Middleware.use('navigation', authGuard);

// Logger runs second (sees only allowed navigations)
Middleware.use('navigation', logger);

Removing Middleware

Middleware.use() returns a dispose function. Call it to remove the middleware from the pipeline: 

const dispose = Middleware.use('search', myMiddleware);

// Later, remove it:
dispose();
⚠ Performance

Every async middleware adds latency. Keep middleware fast, especially on the navigation and page-load pipelines where users notice delays. Profile using performance.mark() if you suspect middleware is the bottleneck.
