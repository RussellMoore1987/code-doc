Docker Containerization

Packaging DevDocs in a Docker container makes deployments reproducible, eliminates "works on my machine" issues, and integrates cleanly with container orchestration platforms like Kubernetes and AWS ECS.

ℹ Static Site Advantage

Because DevDocs consists entirely of static files, the Docker image is extremely small — just an Nginx or Caddy base image plus your project files. No runtime, no database, no long-running process other than the web server.

Writing the Dockerfile

# Dockerfile
FROM nginx:1.27-alpine

# Remove default Nginx content
RUN rm -rf /usr/share/nginx/html/*

# Copy project files
COPY . /usr/share/nginx/html/

# Copy custom Nginx config (optional but recommended)
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]

Nginx Configuration

# nginx.conf
server {
  listen 80;
  server_name _;
  root /usr/share/nginx/html;
  index index.html;

  # Security headers
  add_header X-Content-Type-Options nosniff always;
  add_header X-Frame-Options DENY always;
  add_header Referrer-Policy strict-origin-when-cross-origin always;

  # Cache static assets
  location ~* \.(css|js|png|jpg|svg|woff2)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
  }

  # No cache for HTML
  location ~* \.html$ {
    add_header Cache-Control "no-cache";
  }

  location / {
    try_files $uri $uri/ /index.html;
  }
}

Building and Running

# Build the image
docker build -t devdocs:latest .

# Run locally on port 8080
docker run -p 8080:80 devdocs:latest

# Open in browser
open http://localhost:8080

Tagging for Production

# Tag with version
docker build -t devdocs:1.4.2 -t devdocs:latest .

# Tag for a registry
docker tag devdocs:latest registry.example.com/devdocs:latest

# Push
docker push registry.example.com/devdocs:latest

Docker Compose

Use Docker Compose for local development and multi-container setups (e.g., docs + reverse proxy): 

# docker-compose.yml
version: "3.9"
services:
  docs:
    build: .
    ports:
      - "8080:80"
    volumes:
      # Mount pages/ for hot-reload during development
      - ./pages:/usr/share/nginx/html/pages:ro
    restart: unless-stopped
# Start
docker compose up -d

# Stop
docker compose down

Multi-Stage Builds

If you add a build step (e.g., a script that pre-processes pages or generates a search index), use a multi-stage build to keep the final image small: 

FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build   # generates /app/dist

FROM nginx:1.27-alpine
COPY --from=builder /app/dist /usr/share/nginx/html/
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]

Container Health Check

# Add to Dockerfile
HEALTHCHECK --interval=30s --timeout=5s --start-period=5s --retries=3 \
  CMD curl -f http://localhost/ || exit 1

Minimizing Image Size

  • Use Alpine-based base images (nginx:alpine is ~7 MB).
  • Add a .dockerignore to exclude development files.
  • Avoid installing unnecessary tools (curl, vim) in the final stage.
# .dockerignore
.git
node_modules
*.md
.env*
.DS_Store
