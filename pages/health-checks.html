Health Checks & Monitoring

Health checks verify that your DevDocs deployment is up and serving content correctly. Combined with monitoring and alerting, they allow you to detect and respond to outages before users report them.

ℹ Static Site Characteristics

Static sites like DevDocs have a narrow failure surface — there is no application server to crash, no database to become unavailable. The main failure modes are: the CDN/host goes down, a deploy corrupts files, or DNS stops resolving. Design your health checks accordingly.

Health Check Endpoint

Add a dedicated health.json file at the root of your project. This file is lightweight, cacheable, and unambiguous — monitoring tools request it to confirm the site is reachable. 

// health.json
{
  "status": "ok",
  "version": "2.4.1",
  "timestamp": "2026-04-22T00:00:00Z"
}

Update this file as part of your build/deploy pipeline so the version always reflects the live deployment.

Uptime Monitoring

UptimeRobot (Free Tier)

  1. Create a new monitor at uptimerobot.com.
  2. Choose HTTP(s) monitor type.
  3. Set the URL to https://docs.example.com/health.json.
  4. Check interval: 5 minutes.
  5. Configure email or Slack alerts for downtime events.

Pingdom / Datadog Synthetics

For enterprise monitoring, configure a synthetic transaction test that:

  1. Loads the homepage (/).
  2. Verifies the <title> tag contains your project name.
  3. Checks that app.js returns HTTP 200.
  4. Validates a known page fragment (pages/intro.html) is accessible.

CDN Health Verification

After every deployment, run a smoke test to confirm the CDN is serving the new version: 

#!/bin/bash
# smoke-test.sh

EXPECTED_VERSION="2.4.1"
HEALTH_URL="https://docs.example.com/health.json"

ACTUAL_VERSION=$(curl -sf "$HEALTH_URL" | python3 -c "import sys,json; print(json.load(sys.stdin)['version'])")

if [ "$ACTUAL_VERSION" != "$EXPECTED_VERSION" ]; then
  echo "ERROR: Expected version $EXPECTED_VERSION but got $ACTUAL_VERSION"
  exit 1
fi

echo "Health check passed: version $ACTUAL_VERSION"

Container Health Checks

For Docker deployments, declare a HEALTHCHECK instruction in your Dockerfile: 

HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
  CMD wget -qO- http://localhost/health.json || exit 1

Kubernetes readiness and liveness probes:

# k8s deployment.yaml
livenessProbe:
  httpGet:
    path: /health.json
    port: 80
  initialDelaySeconds: 5
  periodSeconds: 30

readinessProbe:
  httpGet:
    path: /health.json
    port: 80
  initialDelaySeconds: 2
  periodSeconds: 10
