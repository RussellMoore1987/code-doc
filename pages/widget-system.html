Widget System

The DevDocs widget system provides a flexible framework for creating reusable, interactive components that can be embedded throughout your documentation. Widgets enhance user experience with dynamic content and interactive elements.

🧩 Widget Philosophy

Widgets are self-contained components that encapsulate functionality, styling, and behavior. They can be easily embedded in any page and configured through data attributes or JavaScript APIs.

Built-in Widgets

DevDocs includes several built-in widgets that cover common documentation needs:

Custom Widget Development

Create custom widgets by extending the base Widget class and implementing the required lifecycle methods: 

class CustomWidget extends DevDocsWidget {
  constructor(element, options = {}) {
    super(element, options);
    this.name = 'custom-widget';
    this.version = '1.0.0';
  }

  init() {
    // Widget initialization logic
    this.setupEventListeners();
    this.loadData();
    this.render();
  }

  render() {
    // Render widget content
    this.element.innerHTML = this.template();
    this.bindEvents();
  }

  template() {
    return `
      <div class="custom-widget-content">
        <h3>${this.options.title}</h3>
        <div class="widget-body">
          ${this.renderContent()}
        </div>
      </div>
    `;
  }

  destroy() {
    // Cleanup logic
    this.removeEventListeners();
    this.element.innerHTML = '';
  }
}

Widget Registration

Register custom widgets with the widget system for automatic initialization: 

// Register the widget
DevDocs.widgets.register('custom-widget', CustomWidget);

// Auto-initialize widgets on page load
document.addEventListener('DOMContentLoaded', () => {
  DevDocs.widgets.initializeAll();
});

// Initialize specific widgets
const widgets = document.querySelectorAll('[data-widget="custom-widget"]');
widgets.forEach(el => new CustomWidget(el));

const chartWidget = new ChartWidget(element, {

Widget Lifecycle

Understanding the widget lifecycle helps in creating robust, performant widgets:

  1. Discovery - Widget elements are found via selectors
  2. Instantiation - Widget class is instantiated with configuration
  3. Initialization - Widget sets up initial state and DOM
  4. Rendering - Widget renders content and becomes interactive
  5. Updates - Widget responds to data changes and user interactions
  6. Cleanup - Widget removes event listeners and cleans up resources

Lifecycle Hooks

class SmartWidget extends DevDocsWidget {
  beforeInit() {
    // Called before initialization
    this.validateConfig();
  }

  afterInit() {
    // Called after initialization
    this.trackInitialization();
  }

  beforeRender() {
    // Called before each render
    this.prepareRenderData();
  }

  afterRender() {
    // Called after each render
    this.announceAccessibility();
  }

  beforeDestroy() {
    // Called before cleanup
    this.saveState();
  }
}

Widget Best Practices

  • Accessibility First - Ensure widgets work with screen readers and keyboards
  • Performance Conscious - Lazy load heavy widgets and optimize rendering
  • Responsive Design - Widgets should adapt to different screen sizes
  • Error Handling - Gracefully handle API failures and invalid data
  • State Management - Maintain consistent state across widget interactions
  • Event Cleanup - Properly remove event listeners to prevent memory leaks
⚡ Performance Tip

Use intersection observers to lazy-load widgets that aren't immediately visible, and implement debouncing for widgets that respond to scroll or resize events to maintain smooth performance.
